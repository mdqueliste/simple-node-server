Text Shadows

The text-shadow property adds shadows to text. It accepts a comma-separated list of shadows to be applied to the text.

/*Syntax - Do not copy*/
text-shadow: [h-shadow] [v-shadow] [blur-radius] [color];

Text-shadow property values

Value Description
h-shadow Required. The position of the horizontal shadow. A negative value places the shadow to the left of the text.
v-shadow Required. The position of the vertical shadow. A negative value places the shadow above the text.
blur-radius Optional. Default value is 0. The higher this value, the bigger the blur; the shadow becomes wider and lighter.
color Optional. The color of the shadow.

Example

Not-so-appealing shadow

<h1 class="ex1">Not-so-appealing shadow</h1>
.ex1{
    text-shadow: 2px 2px;
}

Example

Shadow with color

<h1 class="ex2">Shadow with color</h1>
.ex2{
    text-shadow: 3px 3px purple;
}

Example

Shadow with blur

<h1 class="ex3">Shadow with blur</h1>
.ex3{
    text-shadow: 2px 2px 5px purple;
}

Example

Multiple shadows

<h1 class="ex4">Multiple shadows</h1>
.ex4{
    color: white;
    text-shadow: 1px 1px 2px black, 0 0 25px blue, 0 0 5px darkblue;
}

Example

Love, Mom

<h1 class="ex5">Love, Mom</h1>
.ex5{
    background-color: #3a50d9;
    color: #e0eff2;
    font: italic bold 90px Georgia, Serif;
    text-shadow: -4px 3px 0 #3a50d9, -14px 7px 0 #0a0e27;
}

Box Shadows

The CSS3 box-shadow property applies shadow to elements. The syntax of box-shadow is almost the same with text-shadow's except for an additional value [spread]. The spread radius is optional. Positive values increase the size of the shadow, negative values decrease the size. Default is 0.

/*Syntax - Do not copy*/
box-shadow: [h-shadow] [v-shadow] [blur-radius] [spread] [color];

Example

Plain shadow

<h1 class="ex1 ex6">Plain shadow</h1>
.ex6{
    border:1px solid black;
    box-shadow: 10px 10px;
}


For the best possible browser support, you should prefix with -webkit- and -moz-:

Example

Shadow with color

<h1 class="ex2 ex7">Shadow with color</h1>
.ex7{
    border:1px solid black;
    
    -webkit-box-shadow: 10px 10px gray; /* Safari 3-4, iOS 4.0.2 - 4.2, Android 2.3+ */
    -moz-box-shadow: 10px 10px gray; /* Firefox 3.5 - 3.6 */
    box-shadow: 10px 10px gray; /* Opera 10.5, IE 9, Firefox 4+, Chrome 6+, iOS 5 */
}

Example

Shadow with blur

<h1 class="ex3 ex8">Shadow with blur</h1>
.ex8{
    border:1px solid black;
    box-shadow: 10px 10px 10px gray;
}

Example

Multiple shadows

<h1 class="ex4 ex9">Multiple shadows</h1>
.ex9{
    border:1px solid black;
    box-shadow: 8px 5px 6px -6px darkblue, 8px 8px 6px -6px blue;
}

Example

Love, Mom

<h1 class="ex5 ex10">Love, Mom</h1>
/*To display an inner shadow, add inset before the other values*/
.ex10{
    border:1px solid black;
    box-shadow: inset 0 0 10px #000000;
}

Border Radius

You can give any element "rounded corners" by applying a border-radius through CSS.

Example

<div id="ex11" class="box-1"></div>
/*Box properties for all the examples in border-radius*/
.box-1{
    width:100px;
    height:100px;
    background-color:#BADA55;
}
#ex11{
    border-radius: 10px;
}

If the element has an image background, it will be clipped at the rounded corner naturally:

Example

<div id="ex12" class="box-1"></div>
#ex12{
    border-radius: 20px;
    background: url(http://wdl.tmimgcdn.com/img_articles/14881/site-background-pattern-07.jpg); 
}

With just one value, border-radius will be the same on all four corners of an element. But that need not be the case. You can specifiy each corner separatedly if you wish. You can also specify two or three values:
  • If one value is set, this radius applies to all 4 corners.
  • If two values are set, the first applies to top-left and bottom-right corner, the second applies to top-right and bottom-left corner.
  • Four values apply to the top-left, top-right, bottom-right, bottom-left corner in that order.
  • Three values: The second value applies to top-right and also bottom-left.


Example

<div id="ex13" class="box-1"></div>
#ex13{
    border-radius: 5px 40px 5px;
}

You may also specify the radiuses in which the corner is rounded by. In other words, the rounding doesn't have to be perfectly circular, it can be elliptical. This is done using a slash ("/") between two values.

Example

<div id="ex14" class="box-1"></div>
<div id="ex15" class="box-1"></div>
#ex14{
    border-radius: 10px/30px; /* horizontal radius / vertical radius */
}
#ex15{
    border-radius: 30px/10px; /* horizontal radius / vertical radius */
}

You may specify the value of border-radius in percentages. This is particularly useful when wanting to create a circle or elipse shape, but can be used any time you want the border radius to be directly correlated with the elements width.

Example

<div id="ex16" class="box-1"></div>
<div id="ex17" class="box-1"></div>
#ex16{
    border-radius: 50%;
}
#ex17{
    border-radius: 50%;
    width:200px;
}


Gradients

CSS3 gradients let you display smooth transitions between two or more specified colors.

Earlier, you had to use images for these effects. However, by using CSS3 gradients you can reduce download time and bandwidth usage. In addition, elements with gradients look better when zoomed, because the gradient is generated by the browser.

Browsers support two types of gradients: linear, defined with the linear-gradient() function, and radial, defined with radial-gradient().

Linear Gradients

To create a linear gradient, you set a starting point and a direction (specified as an angle) along which the gradient effect is applied. You also define color stops. Color stops are the colors you want Gecko to render smooth transitions among, and you must specify at least two of them, but can specify more to create more complex gradient effects.

Simple linear gradients

Example

<div id="grad1" class="box-2"></div>
/*Box properties for all the examples in gradients*/
.box-2{
    width:100px;
    height:100px;
    border:1px solid black;
}
#grad1{
    /*creates a horizontal gradient fading from blue to white*/
    background: linear-gradient(to bottom, blue, white); 
}

Example

<div id="grad2" class="box-2"></div>
#grad2{
	/*creates a gradient fading from left to right*/
    background: linear-gradient(to right, blue, white); 
}

You can make the gradient run diagonally by specifying both the horizontal and vertical starting positions.

Example

<div id="grad3" class="box-2"></div>
#grad3{
     /*creates a diagonal gradient*/
    background: linear-gradient(to bottom right, blue, white);
}

Using angles

If you don't specify an angle, one is determined automatically based on the given di v rection. If you'd like more control over the direction of the gradient, you can set the angle specifically.

Example

<div id="grad4" class="box-2"></div>
#grad4{
    /*creates a gradient with 70 degrees*/
    background: linear-gradient(70deg, black, white); 
}
The angle is specified as an angle between a horizontal line and the gradient line, going counter-clockwise. In other words, 0deg creates a vertical gradient from the bottom to the top, while 90deg generates a left to right horizontal gradient.

Color Stops

At a minimum, gradient backgrounds will transition from one color to another; however, we may add multiple colors to a gradient and have the browser transition between all of them. To do this we’ll add color stops to the given gradient function, with commas separating each color stop from the next.

Example

<div id="grad5" class="box-2"></div>
#grad5{
    background: linear-gradient(to right, #f6f1d3, #648880, #293f50);
}

By default, the browser will position every color stop an equal distance from the next and will transition between them accordingly. If more control over how colors are positioned is desired, a location along the gradient may be identified for each color stop. The location should be declared as a length value and should fall after the color value.

Example

<div id="grad6" class="box-2"></div>
#grad6{
    background: linear-gradient(to right, #f6f1d3, #648880 85%, #293f50);
}
Unless specified otherwise, the first color stop will be positioned at 0%, and the last color stop will be positioned at 100%.

Transparency and gradients

Gradients support transparency. You can use this, for example, when stacking multiple backgrounds, to create fading effects on background images.

Example

<div id="grad7" class="box-2"></div>
#grad7{
    background: linear-gradient(to right, rgba(255,255,255,0), rgba(255,255,255,1)), url('http://coolchaser-static.coolchaser.com/images/themes/t/411346-i326.photobucket.com-albums-k436-JesusBear3-blue_black_swirljpeg.jpg');
}
The backgrounds are stacked with the first specified background on top, and each successive background farther away. By stacking backgrounds this way, you can create very creative effects as seen above.

Radial gradients

While the linear gradient is perfect for a gradient moving from one direction to another, often the need for a radial gradient arises. Radial background gradients work just like linear gradients and share many of the same values. For radial gradients, instead of using the linear-gradient() function within the background or background-image property, we’ll use the radial-gradient() function.

Example

<div id="grad8" class="box-2"></div>
#grad8{
    background: radial-gradient(#648880, #293f50);
}

Radial gradients work from the inside to the outside of an element. Thus, the first color identified within the radial-gradient() function will sit in the absolute center of the element, and the second color will sit on the outside of an element. The browser will then create the transition between the two colors.

Web Fonts with @font-face

Using Web-safe Fonts

By default there are a few fonts that are pre-installed on every computer, tablet, smart-phone, or other web-browsing-capable device. Because they’ve been installed on every device, we can use these fonts freely within our websites, knowing that no matter what device is browsing our site, the font will render properly. These fonts have become known as “web-safe fonts.” There are only a handful of them, and the safest of the web-safe fonts are listed here:
  • Arial
  • Courier New, Courier
  • Garamond
  • Georgia
  • Lucida Sans, Lucida Grande, Lucida
  • Palatino Linotype
  • Tahoma
  • Times New Roman, Times
  • Trebuchet
  • Verdana

Embedding Web Fonts

We also have the ability to upload fonts to a server and include them on a website via the CSS @font-face at-rule. This capability has done wonders for online typography.

First, we use the @font-face at-rule to identify our font’s name, via the font-family property, as well as the source of our font (the path to the font file containing our chosen font), via the src property. From there we are able to use this font by including its name within any font-family property value.

Example

This is Bitstream Vera Serif.

<h2 id="font1">This is Bitstream Vera Serif.</h2>
@font-face {
	font-family: "Bitstream Vera Serif";
	src: url("https://mdn.mozillademos.org/files/2468/VeraSeBd.ttf");
}
#font1{
	font-family: "Bitstream Vera Serif", serif;
}

Different Font formats

  • TrueType Fonts (TTF)
  • OpenType Fonts (OTF)
  • The Web Open Font Format (WOFF)
  • The Web Open Font Format (WOFF 2.0)
  • SVG Fonts/Shapes
  • Embedded OpenType Fonts (EOT)

Fortunately, the value of using new typefaces online has been recognized, and companies have begun developing ways to license and include new fonts on websites. Google Fonts license the fonts for free.

Let’s head over to the Google Fonts website and search for the font we’d like to use, 'Indie Flower'. Google will give us an additional <link> element to include in the <head> element of all of our pages. The new element will include the proper style sheet reference to Google, which will take care of including a new CSS file with the proper @font-face at-rule necessary for us to use our desired font.

Example

This is Indie Flower.

<h2 id="font2">This is Indie Flower.</h2>
#font2{
	font-family: "Indie Flower", cursive;
}

For the above example to work, you should make sure to add the following in your <head> section: 
<link href='http://fonts.googleapis.com/css?family=Indie+Flower' rel='stylesheet' type='text/css'>
Now try to add another font you like from Google fonts!
