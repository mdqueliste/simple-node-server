In HTML, there is another classification of elements: inline or block.

Inline Element

An inline element is an element that lies on the line with no subsequent break, unless the line has come to the end of available browser space.

Example

<img src="http://www.chykwest.com/wp-content/uploads/family-tree.jpg" width="100">
<img src="http://lakelandtn.gov/images/pages/N147/tree.png" width="100">
Subsequent images in the above example were displayed side-by-side. No break after each image.
Other inline elements: <a>, <span>

Block level Element

A block level element defines a block, and there is a break between it and the next element.

Example

First paragraph

Second paragraph 

<p> First paragraph </p>
<p> Second paragraph </p>
Other inline elements: <h1>, <table>

Knowing whether an element is an inline element or a block element will help in positioning the element in a web page. There are different properties to position an element like the float and position properties.

Float Property

The float CSS property specifies that an element should be taken from the normal flow and placed along the left or right side of its container, where text and inline elements will wrap around it.

Float Property Behavior
  • Texts and other elements will continue to flow around the floated element. (but not divs! Divs following a floated element will stay behind the floated element. More on this later.)
  • Try adding a new heading after the code of the above example, say, another type of cactus. See what happens: 
    <h4>Columnar Cactus</h4>

    Notice that the new heading still flows around the floated barrel cactus image which may not be what we want. To turn-off floats, we can use the clear property. The clear property specifies on which sides of an element floating elements are not allowed. It's values are: left, right, both.

    Try changing the Columnar Cactus heading to something like this and add the following css: 
    <h4 class="to-clear-left">Columnar Cactus</h4>
    .to-clear-left{
    clear:left;
}
    See that the Columnar Cactus heading turned the float off and went back to the normal flow.

  • For example, we want to create columns in our page so we float two divs to the opposite sides. The problem here is that the divs will continue to spread apart as the browser size increases, and come together as the browser size decreases. Try resizing your browser window and see what happens to the following example:

CSS Positioning

The position property allows positioning of HTML elements in a variety of ways. It can also place an element behind another. Types:
  1. Relative Positioning
  2. Absolute positioning
  3. Fixed Positioning

Example

One
Two
Three
<div>
	<div class="box">One</div>
	<div class="box">Two</div>
	<div class="box">Three</div>
</div>
.box {
	background-color: gray;
	width: 100px;
	height: 100px;
	margin: 5px;
}
Note: You may use the class to-clear-left again in the div containing the boxes.

Relative Positioning

A relative positioned element is positioned relative to its normal position. The box is shifted from this normal position according to the properties — top or bottom and/or left or right.

The position properties of the three boxes are not set. Take note of their normal position above. Now, let us try set the box Two to position relative.

Change the class of box Two in your HTML:

<div>
	<div class="box"></div>
	<div class="pos-relative"></div>
	<div class="box"></div>
</div>


Also, add the following to your CSS code:

.pos-relative {
	background-color: #d6d7d8;
	width: 100px;
	height: 100px;
	margin: 5px;
	
	/*To properly position an element using relative, top or bottom (choose one) 
	and/or left or right (choose one) properties should be used.*/
	position:relative;
	top:20px;
	right:20px;
}

Example

One
Two
Three

Notice that from box Two's normal position, it was offset 20px down (caused by top:20px;) and 20px to the left (caused by right 20px). This means that from it's original position, 20px was added to the top edge which caused the box to move down. Same with adding 20px to the right edge causing it to move left.

top, bottom, left, and right can have negative values which will cause the reverse effect of the positive values.

Absolute Positioning

An absolutely positioned element:
  • is removed from normal flow
  • is positioned in relation to
    • a positioned container element
    • the root element HTML (if no positioned container was found)
Again modify your HTML:

<div class="pos-container">
	<div class="box"></div>
	<div class="pos-absolute"></div>
	<div class="box"></div>
</div>


Also, add the following to your CSS code:

.pos-container{
	position:relative;
	
}
.pos-absolute {
	background-color: #d6d7d8;
	width: 100px;
	height: 100px;
	margin: 5px;
	
	/*To properly position an element using absolute, top or bottom (choose one) 
	and/or left or right (choose one) properties should be used.*/
	position:absolute;
	top:20px;
	right:20px;
}

Example

One
Two
Three

Different from the effect of relative positioning, absolute positioning placed box Two relative to its positioned parent. In the code above, we have added a class pos-container and set it to be positioned relative. This div is the container of our boxes. Since box Two is set to absolute, it looked for a positioned container and was offset from the edges of that container.

Notice too that in relative positioning, the space for box Two was preserved while in absolute positioning, the space was occupied by the following element.

Fixed Positioning

Fixed positioning is a subcategory of absolute positioning. The only difference is that a fixed positioned element is fixed with respect to the browser's viewport and does not move when scrolled.

Change the class of box Two for the last time:

<div class="pos-container">
	<div class="box"></div>
	<div class="pos-fixed"></div>
	<div class="box"></div>
</div>


Change the class and the position value to fixed and see what happens:

.pos-fixed {
	background-color: #d6d7d8;
	width: 100px;
	height: 100px;
	margin: 5px;
	
	/*To properly position an element using fixed, top or bottom (choose one) 
	and/or left or right (choose one) properties should be used.*/
	position:fixed;
	top:20px;
	right:20px;
}

Overlapping Elements

When elements are positioned outside the normal flow, they can overlap other elements. The z-index property specifies the stack order of an element (which element should be placed in front of, or behind, the others). An element can have a positive or negative stack order.

An element with greater stack order is always in front of an element with a lower stack order.
Note: If two positioned elements overlap without a z-index specified, the element positioned last in the HTML code will be shown on top.
