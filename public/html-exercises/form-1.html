HTML Forms

HTML Forms are one of the main points of interaction between a user and a web site or application. They allow users to send data to the web site. Most of the time that data is sent to the web server, but the web page can also intercept it to use it on its own.

The <form> Element

All HTML forms start with a <form> element like this: 
<form>
<!--Insert form elements-->
 </form>
HTML forms contain form elements.
Form elements are different types of input elements, checkboxes, radio buttons, submit buttons, and more.

Form Elements

Element Description
input specifies an input field where the user can enter data
label associates caption to a control
select used to create a drop-down list
option represents an item in a drop-down list
textarea represents a multi-line plain-text editing control
fieldset used to group several controls within a web form
legend represents a caption for the content of its parent fieldset element
button represents a clickable button

Input type: text

The <input> element is the most important form element. The type attribute defines the way the <input> element behaves. It can radically change the element so pay attention to it. The most common type value is text.

Example

<label> Username: <input type="text" name="username"></label>
<!--OR--> 
<label for="username"> Username: </label><input type="text" name="username" id="username">
Notice that the <input> tag is an auto-closing element, which means that it does not need a closing </input> tag.

The <label> element does not render as anything special for the user. However, it provides a usability improvement for mouse users, because if the user clicks on the text within the <label> element, it toggles the control.

A label can be bound to an element either by placing the element inside the <label> element or by using the for attribute.

Input type: password

Example

<label> Password: <input type="password" name="password" required></label>
The above two input elements look the same but the difference can be seen when you try to type inside the password field. Bullets will be displayed instead of characters. This is caused by the type attribute with value password which made the browser recognize that the input to the field should not be displayed as is.

An additional attribute is used here which is new in HTML5, the required attribute. It specifies that an input field must be filled out before submitting the form.

Input type: radio

Example

Status: 
Status:
<label><input type="radio" name="status" value="single"> Single </label>
<label><input type="radio" name="status" value="married"> Married </label>
We use type="radio" here for all options that belongs to the item Status. It is important to take note that when using this type, the name attribute of all the options should be the same (here, all items have the name value "status"). This tells the browser not to allow the user to choose more than one of the options.

The select element

For form elements which would require many options, a drop-down list may be used. This allows multiple-option items to occupy minimal space in the page.

Example

<label>
Course:
<select name="course">
	<option value="cmsc">BSCS</option>
	<option value="devcom">BSDC</option>
	<option value="comarts">BACA</option>
	<option value="nutri">BSN</option>
	<option value="others">Others</option>
</select>
</label>
The <select> element wraps the options to be displayed in the drop-down list. Each option is described by an <option> element.

The textarea element

This is the element to represent a multi-line plain-text editing control.

Example

<label> Motto in life: <br><textarea rows="10" cols="50">Type something here..</textarea></label>
You can specify the width and height of the textarea using the attributes rows and cols.

Fieldset

To organize forms and group related elements, a <fieldset> tag is used. Let us try to combine some related form elements above and use a fieldset to group them.

Example

Account Details 


<fieldset>
	<legend>Account Details</legend>
	<label> Username: <input type="text" name="username"></label> <br>
	<label> Password: <input type="password" name="password" required></label>
</fieldset>
The <legend> element provides a caption for the content of the fieldset.

Finally, the Button

Our form is almost ready; we just have to add a button to allow the user to send their data once they have filled out the form. This is simply done by using the <button> element:

Example

Account Details 


<button type="submit">Submit</button>
The above example made use of the submit type. If you click the button, the data on the form elements will be submitted to the page to process the data. More on that when you take higher CMSC subjects. For now, our concern is to prepare the HTML form.

If you don't answer the Password field and you click Submit, a tooltip will be displayed. Note that your code should be inside the <form> </form> if you want to try this and make it work.

Button type

The type attribute of a button element can have any of the following values:
Type Description
submit button that submits the form data to the server
reset button that resets all the controls to their initial values
button button that has no default behavior
